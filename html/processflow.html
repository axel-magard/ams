AMS Process Flow

An AMS Process Flow represents the sequence of Operations needed to manufacture parts of a given Product Type.

Each step in a Process Flow ( a transition ) is defined by its predecessor Operation, successor Operation and a transition probability. A predecessor Operation can have multiple successor Operations with different transition probabilities. The sum of these can not exceed 100%, but can be smaller than 100% to indicate that parts are lost (scrapped) when leaving the predecessor operation.

A step in the Process Flow ( or transitionn ) following attributes:

  • Predecessor Operation: Operation parts are coming from
  • Successor Operations: Operations parts are going to
  • Probability: Probability parts are taking this path from predecessor Operation to successor Operation

In addition the following cost and resource specific attributes can be defined:

  • Operation Time: Operator time needed to transport part from predecessor Operation to successor Operation
  • Transition Cost: Cost occuring for this transport
  • Transition Time: Time needed for this transport

To define a Process Flow right-click on a Product Type in the tree view of model artefacts on the left side of the screen and select "Edit Process Flow".

→

The "Edit Process Flow" dialog also allows to add or delete successor Operations for a selected predecessor Operation.
